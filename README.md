|---|---|---|
| BPE Tokenizer + Embeddings | Continuous Semantic Encoder (CSE) | No vocabulary, any UTF-8, language-agnostic |
| Weight matrices | Resonance Field (sparse 3D tensor) | Local updates, zero catastrophic forgetting |
| Attention O(n²) | Gravitational Relevance O(log n) | Scales to 16k+ tokens on same GPU |
| Backpropagation | Thermodynamic Settling | Real-time single-sample learning |
| Neurons / Layers | Causal Geometry Nodes | Stores WHY not just WHAT |
| Context window | Three-Tier Memory | Permanent cross-session retention |
| Static model file (.pt) | Living Field Snapshot (.flx) | Continues learning on reload |
| Per-user database | Personal Fabric | Sovereign field, data stays on device |

## The .flx File Format

FLUX models are saved as `.flx` files — PyTorch saves with a deliberate schema capturing the full **living state**, not just weights:

```python
{
    'format': 'FLUX',
    'can_continue_learning': True,   # Always true — FLUX never "freezes"

    # Neural weights (standard state_dicts)
    'cse_state':    ...,   # Phase 1 — frozen forever after training
    'cwc_state':    ...,   # Phase 1.5
    'field_state':  ...,   # Phase 2/2.5 sparse registry

    # Physics state — NOT captured by state_dict, unique to FLUX
    'field_mass':          tensor,   # Evidence mass per field location
    'attractor_catalog':   [...],    # What is known and where
    'impl_store':          {...},    # Causal arrows (A implies B)
    'field_step_count':    int,      # Exact step where learning left off

    # Config — enough to reconstruct the full architecture
    'field_config': {'h': 64, 'w': 64, 'd': 64, 'features': 512},
    'cse_config':   {'wave_dims': {...}},
}
```

A `.flx` loaded mid-deployment continues learning immediately. Loading and inference are the same operation — the field is always in an active state.

**Upgrading is additive:**
```
flux_v1.flx  ──► flux_v2.flx  ──► flux_v3.flx
  P1–P2.5         + more data        + Phase 3
                  field grew         field grew more

Each version is a strict superset. Nothing is ever forgotten.
```

## Key Invariants

1. Every phase saves a checkpoint before ending
2. Every phase loads and verifies the previous checkpoint at start
3. Every phase has at least two demo scripts and three test scripts
4. Every phase produces `RESULTS_PHASE_N.md`
5. No phase assumes a clean state — always load and verify
6. Every phase has a Kaggle notebook and uploads to HuggingFace
7. Every phase logs via `PhaseLogger` to `logs/phaseN.log`
8. **The CSE is frozen after Phase 1 — never retrained**
