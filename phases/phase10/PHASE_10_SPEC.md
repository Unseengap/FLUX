## Component 1: FLUXWaveLarge — Scaled Configuration

Phase 10 wraps FLUXLarge + Phase 9 modules into a single coherent class
for training and inference. No new architecture — just the integration layer.

```python
class FLUXWaveLarge(nn.Module):
    """
    Complete wave-level FLUX model for scaled training.

    Combines:
    - FLUXLarge base (CSE, field, GR, TL, CGN, memory, bridges)
    - WaveChunker (boundary detection)
    - WaveGenerator (concept-level sequence prediction)
    - WaveToText (last-mile spelling)
    - ThermodynamicWaveSampler (non-parametric)

    This is the full FLUX model that will be benchmarked against GPT-2.

    Unlike Phase 8's FLUXLarge which bolted a byte-level GRU on top,
    FLUXWaveLarge generates at the wave level — the right abstraction.

    Args:
        base_model: FLUXLarge with all Phases 1-7 components
        wave_chunker: Phase 9 WaveChunker
        wave_generator: Phase 9 WaveGenerator
        wave_to_text: Phase 9 WaveToText
        device: Target device
    """

    def __init__(
        self,
        base_model: FLUXLarge,
        wave_chunker: WaveChunker,
        wave_generator: WaveGenerator,
        wave_to_text: WaveToText,
        device: str = 'cpu',
    ):
        super().__init__()
        self.base = base_model
        self.wave_chunker = wave_chunker
        self.wave_generator = wave_generator
        self.wave_to_text = wave_to_text
        self.wave_sampler = ThermodynamicWaveSampler()
        self._device_str = device

    @classmethod
    def from_phase9_checkpoint(cls, device: str = 'cpu') -> 'FLUXWaveLarge':
        """Load complete model from Phase 9 checkpoint."""
        components = load_phase9_for_scaling(device=device)
        return cls(
            base_model=components['model'],
            wave_chunker=components['wave_chunker'],
            wave_generator=components['wave_generator'],
            wave_to_text=components['wave_to_text'],
            device=device,
        )

    def generate(
        self,
        prompt: str,
        max_waves: int = 30,
        temperature: float = 0.8,
    ) -> str:
        """
        Generate text from prompt using wave-level generation.

        Args:
            prompt: Input text
            max_waves: Maximum waves (concepts) to generate
            temperature: Sampling temperature (higher = more creative)

        Returns:
            Generated text string
        """
        # 1. Encode prompt
        wave_seq, wave_vec, field_context = self.base._get_context(prompt)

        # 2. Chunk prompt into word-level waves
        prompt_chunks, _ = self.wave_chunker(wave_seq)

        # 3. Generate new waves (thinking in concepts)
        generated_waves, confidences = self.wave_generator.generate(
            field_context=field_context,
            max_waves=max_waves,
            flux_model=self.base,
        )

        # 4. Spell each wave into text (last-mile)
        text_parts = []
        for wave, conf in zip(generated_waves, confidences):
            sampled = self.wave_sampler.sample_wave(wave, conf)
            chunk_bytes = self.wave_to_text.decode(sampled, temperature=temperature)
            text_parts.append(chunk_bytes.decode('utf-8', errors='replace'))

        return prompt + ' '.join(text_parts)

    def learn_fact(self, fact: str) -> None:
        """One-shot fact learning via episodic memory."""
        self.base.learn_fact(fact)

    def query(self, question: str) -> str:
        """Query episodic memory for a learned fact."""
        return self.base.query(question)

    def parameter_count(self) -> Dict[str, int]:
        """Return parameter counts by component."""
        return {
            'base_model': sum(p.numel() for p in self.base.parameters()),
            'wave_chunker': sum(p.numel() for p in self.wave_chunker.parameters()),
            'wave_generator': sum(p.numel() for p in self.wave_generator.parameters()),
            'wave_to_text': sum(p.numel() for p in self.wave_to_text.parameters()),
            'total': sum(p.numel() for p in self.parameters()),
        }
```

## Component 2: Curriculum Learning — Simple to Complex

Phase 8 just streamed OpenWebText in random order. Phase 10 uses curriculum
learning to help the wave-level model build capability progressively.

### Why Curriculum Matters for Wave-Level

WaveGenerator predicts the next **concept**, not the next byte. Simple text
(short sentences, common words) has shorter wave sequences with clearer
attractor neighborhoods. Complex text (long sentences, rare vocabulary,
technical writing) has longer wave sequences with ambiguous boundaries.

Training on simple text first lets:
- WaveChunker learn clean word boundaries before handling edge cases
- WaveGenerator learn common concept transitions before rare ones
- WaveToText master frequent words before encountering unusual spellings
- The field build strong, well-separated attractors before dense regions

```python
class CurriculumScheduler:
    """
    Progressive data ordering for wave-level training.

    Stages:
        Stage 1 (0-25%):   Short texts (< 200 chars), common vocabulary
        Stage 2 (25-60%):  Medium texts (< 500 chars), broader vocabulary
        Stage 3 (60-85%):  Full-length texts, all vocabulary
        Stage 4 (85-100%): Hard examples — long, technical, rare words

    Scoring heuristics:
        - Shorter text = easier (fewer wave steps)
        - More common words = easier (stronger attractors)
        - Simpler grammar = easier (more predictable wave transitions)
    """

    def __init__(self, total_steps: int):
        self.total_steps = total_steps
        self.thresholds = [
            (0.25, 200, 'short'),    # first 25%: texts < 200 chars
            (0.60, 500, 'medium'),   # next 35%: texts < 500 chars
            (0.85, 2000, 'full'),    # next 25%: texts < 2000 chars
            (1.00, 99999, 'hard'),   # final 15%: everything
        ]

    def max_text_length(self, current_step: int) -> int:
        """Return maximum text length for current training progress."""
        progress = current_step / max(self.total_steps, 1)
        for threshold, max_len, _ in self.thresholds:
            if progress <= threshold:
                return max_len
        return 99999

    def difficulty_score(self, text: str) -> float:
        """
        Score text difficulty (0.0 = easy, 1.0 = hard).

        Factors:
        - Length (longer = harder)
        - Average word length (longer words = harder)
        - Unique word ratio (more unique = harder)
        """
        words = text.split()
        if not words:
            return 0.0

        len_score = min(len(text) / 2000.0, 1.0)
        avg_word_len = sum(len(w) for w in words) / len(words)
        word_len_score = min(avg_word_len / 10.0, 1.0)
        unique_ratio = len(set(w.lower() for w in words)) / len(words)

        return (len_score + word_len_score + unique_ratio) / 3.0
```

## Component 3: Wave-Level Perplexity

Phase 8 measured perplexity at the byte level. Phase 10 needs to measure it
at the wave level AND convert to a byte-level metric for fair comparison
against GPT-2.

### The Problem

GPT-2 perplexity is measured in tokens (BPE subwords). FLUX Phase 8 measured
in bytes. Neither is directly comparable. We need a common metric.

### The Solution: Bits-Per-Character (BPC)

BPC is the standard unit-agnostic metric for language model comparison:

```python
class WavePerplexity:
    """
    Compute perplexity for wave-level FLUX model.

    Two metrics:
    1. Wave perplexity: based on wave prediction loss (internal metric)
    2. BPC (bits per character): universal metric for comparison with GPT-2

    BPC allows direct comparison regardless of whether the model uses
    bytes (FLUX), BPE tokens (GPT-2), or wave chunks (Phase 10).

    BPC formula:
        BPC = total_cross_entropy_loss / (num_characters * ln(2))

    Lower BPC = better model. Typical values:
        GPT-2 small: ~1.0-1.2 BPC
        Good byte-level model: ~1.0-1.1 BPC
        Random prediction: ~8.0 BPC (log2(256))
    """

    @staticmethod
    def compute_wave_perplexity(
        model: 'FLUXWaveLarge',
        texts: List[str],
    ) -> Dict[str, float]:
        """
        Compute wave-level and byte-level perplexity.

        For each text:
        1. Encode with CSE → waves
        2. Chunk into word-level waves
        3. For each chunk position, compute WaveGenerator's predicted
           wave vs actual wave → cosine loss
        4. Decode each wave via WaveToText → byte predictions
        5. Compare decoded bytes vs actual text bytes → cross-entropy

        Returns:
            Dict with wave_ppl, byte_ppl, bpc, samples
        """
        model.eval()
        wave_losses = []
        byte_losses = []
        total_chars = 0

        with torch.no_grad():
            for text in texts:
                if not text or len(text.strip()) < 10:
                    continue

                # Encode and chunk
                wave_seq = model.base.cse.encode(text).full.to(model._device_str)
                chunks, spans = model.wave_chunker(wave_seq)

                if chunks.shape[0] < 2:
                    continue

                # Wave perplexity: predict each chunk from previous + context
                wave_vec = wave_seq.mean(dim=0)
                field_features, _, _ = model.base.field.query(wave_vec, k=4)
                context = field_features.mean(dim=0)

                # Predict waves
                predicted = model.wave_generator.forward_teacher(
                    field_context=context,
                    target_waves=chunks,
                )
                wave_loss = F.mse_loss(predicted, chunks[1:])
                wave_losses.append(wave_loss.item())

                # Byte perplexity: decode each chunk, compare to actual text
                text_bytes = list(text.encode('utf-8', errors='replace'))
                for i, (chunk_wave, (start, end)) in enumerate(zip(chunks, spans)):
                    actual_bytes = text_bytes[start:end]
                    if not actual_bytes:
                        continue
                    target = torch.tensor(actual_bytes, dtype=torch.long,
                                          device=model._device_str)
                    logits = model.wave_to_text(chunk_wave, target)
                    # logits: [chunk_len+1, 257], target: [chunk_len]
                    # Align: first chunk_len logits predict chunk_len bytes
                    byte_loss = F.cross_entropy(
                        logits[:len(actual_bytes)],
                        target,
                    )
                    byte_losses.append(byte_loss.item())
                    total_chars += len(actual_bytes)

        model.train()

        avg_wave_loss = sum(wave_losses) / max(len(wave_losses), 1)
        avg_byte_loss = sum(byte_losses) / max(len(byte_losses), 1)

        return {
            'wave_ppl': math.exp(min(avg_wave_loss, 20.0)),
            'byte_ppl': math.exp(min(avg_byte_loss, 20.0)),
            'bpc': avg_byte_loss / math.log(2),
            'samples': len(wave_losses),
            'total_chars': total_chars,
        }
```

## Training Strategy

### Three-Stage Training

Phase 10 trains in three stages, each building on the previous:

```
Stage 1: Warmup (10% of steps)
    - WaveToText fine-tuning on OpenWebText vocabulary
    - WaveChunker boundary refinement
    - WaveGenerator frozen (preserves Phase 9's learned structure)
    - Field: settling only (no gradient)
    - Purpose: strengthen the spelling and chunking before wave prediction

Stage 2: Main Training (75% of steps)
    - All modules unfrozen: WaveGenerator + WaveToText + bridges
    - Field: thermodynamic settling (no gradient, but learns via TL)
    - Curriculum: short → medium → full-length texts
    - Gradient accumulation: effective batch size 16
    - Mixed precision (fp16)
    - Checkpoint every 5000 steps
    - Purpose: the main training run, learning concept transitions

Stage 3: Refinement (15% of steps)
    - Lower learning rate (10x reduction)
    - Hard examples only (long, technical, rare vocabulary)
    - WaveToText learning rate reduced further (mostly frozen)
    - WaveGenerator still learning
    - Purpose: polish quality on difficult text
```

### Training Loop

```python
class OpenWebTextWaveTrainer:
    """
    Train FLUXWaveLarge on OpenWebText using wave-level objectives.

    Unlike Phase 8's trainer which used byte-level cross-entropy as the
    sole training signal, Phase 10 uses THREE loss signals:

    1. Wave prediction loss (MSE + cosine):
       WaveGenerator predicts waves, compared to CSE ground truth.
       This is the PRIMARY training signal.

    2. WaveToText spelling loss (cross-entropy):
       WaveToText decodes predicted waves, compared to actual bytes.
       This is the SECONDARY signal that keeps spelling accurate.

    3. Interference coherence loss (cosine):
       Consecutive generated waves should have reasonable similarity.
       This prevents the generator from producing random disconnected concepts.

    Total loss = wave_loss + α * spelling_loss + β * coherence_loss
    Default: α = 0.5, β = 0.1

    Args:
        model: FLUXWaveLarge
        lr: Initial learning rate
        grad_accum: Gradient accumulation steps
        checkpoint_interval: Steps between saves
        log: PhaseLogger instance
        max_seq_bytes: Maximum bytes per document
    """

    def __init__(
        self,
        model: 'FLUXWaveLarge',
        lr: float = 3e-4,
        grad_accum: int = 4,
        checkpoint_interval: int = 5000,
        log: Optional[PhaseLogger] = None,
        max_seq_bytes: int = 1024,
        spelling_weight: float = 0.5,
        coherence_weight: float = 0.1,
    ):
        self.model = model
        self.lr = lr
        self.grad_accum = grad_accum
        self.checkpoint_interval = checkpoint_interval
        self.log = log
        self.max_seq_bytes = max_seq_bytes
        self.spelling_weight = spelling_weight
        self.coherence_weight = coherence_weight

        # Separate parameter groups with different learning rates
        self.param_groups = [
            # WaveGenerator: primary training target
            {
                'params': list(model.wave_generator.parameters()),
                'lr': lr,
                'name': 'wave_generator',
            },
            # WaveToText: secondary, lower LR to preserve spelling
            {
                'params': list(model.wave_to_text.parameters()),
                'lr': lr * 0.3,
                'name': 'wave_to_text',
            },
            # WaveChunker: fine-tune boundaries slowly
            {
                'params': list(model.wave_chunker.parameters()),
                'lr': lr * 0.1,
                'name': 'wave_chunker',
            },
            # Bridge projections: allow fine-tuning
            {
                'params': (
                    list(model.base.wave_to_field.parameters()) +
                    list(model.base.field_to_wave.parameters())
                ),
                'lr': lr * 0.1,
                'name': 'bridges',
            },
        ]

        self.optimizer = torch.optim.AdamW(
            self.param_groups, weight_decay=0.01
        )

        # Mixed precision
        self.use_amp = torch.cuda.is_available()
        self.scaler = torch.amp.GradScaler('cuda') if self.use_amp else None

        self._global_step = 0
        self._tokens_seen = 0

    def train_step(self, text: str) -> dict:
        """
        Single training step on one document.

        Pipeline:
        1. CSE encode (no grad) → wave_sequence [seq, 432]
        2. WaveChunker → target_chunks [N, 432], spans
        3. Thermodynamic settle (field learning, no grad)
        4. Get field context (no grad through field query)
        5. WaveGenerator predicts each chunk from context → predicted_waves
        6. WaveToText decodes predicted waves → predicted_bytes
        7. Compute 3 losses → backprop through WaveGenerator + WaveToText
        """
        device = self.model._device_str

        # ── 1. CSE encode (frozen) ──
        with torch.no_grad():
            wave = self.model.base.cse.encode(text)
        wave_seq = wave.full.to(device)
        wave_vec = wave_seq.mean(dim=0)

        # ── 2. Chunk into word-level targets ──
        target_chunks, spans = self.model.wave_chunker(wave_seq)
        if target_chunks.shape[0] < 3:
            return {'loss': 0.0, 'skipped': True}

        # ── 3. Thermodynamic settle (field learns, no backprop) ──
        with torch.no_grad():
            settle_result = self.model.base.tl.settle_once(wave_vec)
            self.model.base._learning_steps += 1
            self.model.base.working_memory.add_perturbation(wave_vec)

        # ── 4. Get FLUX context (no grad through field/CGN) ──
        with torch.no_grad():
            field_features, sims, locs = self.model.base.field.query(
                wave_vec.detach(), k=4
            )
            combined = field_features.mean(dim=0)
            cgn_out = self.model.base.cgn(combined)
            context = (combined + cgn_out).detach()

        # ── 5. Wave prediction (WITH grad) ──
        amp_ctx = torch.amp.autocast('cuda') if self.use_amp else nullcontext()
        with amp_ctx:
            # Predict each chunk from context + previous chunks
            predicted_waves = self.model.wave_generator.forward_teacher(
                field_context=context,
                target_waves=target_chunks,
            )
            # predicted_waves: [N-1, 432] (predicts chunks 1..N from 0..N-1)

            # ── Loss 1: Wave prediction (MSE + cosine) ──
            target = target_chunks[1:]  # [N-1, 432]
            mse_loss = F.mse_loss(predicted_waves, target)
            cos_loss = 1.0 - F.cosine_similarity(
                predicted_waves, target, dim=-1
            ).mean()
            wave_loss = mse_loss + cos_loss

            # ── Loss 2: Spelling (cross-entropy per chunk) ──
            text_bytes = list(text.encode('utf-8', errors='replace'))
            spelling_loss = torch.tensor(0.0, device=device)
            n_spell_chunks = 0
            for i, (pred_wave, (start, end)) in enumerate(
                zip(predicted_waves, spans[1:])
            ):
                actual = text_bytes[start:end]
                if not actual or len(actual) > 20:
                    continue
                byte_target = torch.tensor(
                    actual, dtype=torch.long, device=device
                )
                logits = self.model.wave_to_text(pred_wave, byte_target)
                # Cross-entropy on decoded bytes
                eos_target = torch.cat([
                    byte_target,
                    torch.tensor([257], device=device),  # EOS
                ])
                chunk_loss = F.cross_entropy(
                    logits[:len(eos_target)],
                    eos_target,
                )
                spelling_loss = spelling_loss + chunk_loss
                n_spell_chunks += 1

            if n_spell_chunks > 0:
                spelling_loss = spelling_loss / n_spell_chunks

            # ── Loss 3: Coherence (consecutive wave similarity) ──
            if predicted_waves.shape[0] > 1:
                coherence_loss = 1.0 - F.cosine_similarity(
                    predicted_waves[:-1], predicted_waves[1:], dim=-1
                ).mean()
            else:
                coherence_loss = torch.tensor(0.0, device=device)

            # ── Total loss ──
            total_loss = (
                wave_loss
                + self.spelling_weight * spelling_loss
                + self.coherence_weight * coherence_loss
            )

        # ── Backward ──
        scaled = total_loss / self.grad_accum
        if self.scaler:
            self.scaler.scale(scaled).backward()
        else:
            scaled.backward()

        self._global_step += 1
        if self._global_step % self.grad_accum == 0:
            if self.scaler:
                self.scaler.unscale_(self.optimizer)
                torch.nn.utils.clip_grad_norm_(
                    self.model.parameters(), max_norm=1.0
                )
                self.scaler.step(self.optimizer)
                self.scaler.update()
            else:
                torch.nn.utils.clip_grad_norm_(
                    self.model.parameters(), max_norm=1.0
                )
                self.optimizer.step()
            self.optimizer.zero_grad()

        self._tokens_seen += len(text_bytes)

        return {
            'loss': total_loss.item(),
            'wave_loss': wave_loss.item(),
            'spelling_loss': spelling_loss.item(),
            'coherence_loss': coherence_loss.item() if isinstance(coherence_loss, torch.Tensor) else coherence_loss,
            'n_chunks': target_chunks.shape[0],
            'skipped': False,
        }
```

## Loss Functions

### Primary: Wave Prediction Loss

```python
# Target: CSE chunk waves (ground truth from input text)
# Predicted: WaveGenerator's next-wave predictions
# Combination of magnitude (MSE) and direction (cosine)

mse_loss = F.mse_loss(predicted_waves, target_waves)
cos_loss = 1.0 - F.cosine_similarity(predicted_waves, target_waves, dim=-1).mean()
wave_loss = mse_loss + cos_loss
```

### Secondary: Spelling Loss

```python
# For each predicted wave, decode to bytes via WaveToText
# Compare decoded bytes to actual text bytes
# Standard cross-entropy — same as any character-level model

spelling_loss = F.cross_entropy(decoded_logits, actual_bytes_with_eos)
```

### Tertiary: Coherence Loss

```python
# Consecutive generated waves should be somewhat similar
# Prevents generator from jumping randomly between unrelated concepts
# Light weight (0.1) — enough to smooth, not enough to force repetition

coherence_loss = 1.0 - F.cosine_similarity(waves[:-1], waves[1:], dim=-1).mean()
```

### Total Loss

```python
total = wave_loss + 0.5 * spelling_loss + 0.1 * coherence_loss
```

## FAQ

**Q: Why not just train Phase 9 for longer instead of making Phase 10?**
A: Phase 9's training script is designed for validation — prove the architecture
works. Phase 10 adds: curriculum learning, 3-stage training with different learning
rates per component, mixed precision, resume-safe checkpointing, wave-level perplexity
measurement, and a proper benchmark suite. These are engineering necessities for
large-scale training that don't belong in Phase 9's minimal-viable proof.

**Q: Why unfreeze the field if Phase 9 kept it frozen?**
A: Phase 9 froze everything to isolate the new modules and prove they work.
Phase 10 unfreezes the field because at scale, the field needs to accumulate
knowledge from hundreds of thousands of documents. A frozen field from Phase 7
(trained on a small corpus) cannot represent the full richness of OpenWebText.
Thermodynamic settling is safe — it's local, it doesn't forget, it's what the
field was designed for.

**Q: What if perplexity doesn't improve much?**
A: Then the bottleneck is in CSE — the wave representations themselves may not
be rich enough for fluent generation. That would point to Phase 1 needing more
training, which is a separate research direction. But the hypothesis is that
the field + wave generation pipeline is sound and just needs data.

**Q: Is Phase 8 completely dead?**
A: Yes. Phase 8's WaveDecoder (byte-level GRU) is not loaded, not used, not
referenced in Phase 10. Phase 8 exists only as a historical data point proving
that byte-level generation was the wrong approach. Phase 10 is the real
"scale and benchmark" phase.

**Q: Can this run on Kaggle's free tier?**
A: The Quick Test tier (10k docs, ~30 min) runs easily. The Light tier (50k docs,
~3 hours) fits in a Kaggle session. For Medium/Full training (200k+ docs), use
Colab Pro or a dedicated GPU. The resume-safe checkpointing means you can split
training across multiple Kaggle sessions.

**Q: What's the minimum training for readable output?**
A: Based on Phase 8's trajectory: ~50k documents should produce recognizable English
words (valid word rate ~70%). ~200k should produce coherent sentences (~85-90%).
500k+ is needed to approach GPT-2 quality. Phase 9's wave architecture gives
~5-7x more semantic learning per document, so these thresholds are much lower
than they were for Phase 8.
