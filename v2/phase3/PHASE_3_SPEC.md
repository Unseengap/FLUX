# Phase 3 Spec: Wave Generation (First Words)

## Goal
Given a text prompt, predict the next sequence of waves from field context,
then decode those waves back to text. This is when FLUX first **generates** —
it stops being a recorder and becomes a speaker.

## Prerequisites
- `checkpoints/phase1_v2.phase.pt` ✓ (CSE + WaveChunker + WaveToText)
- `checkpoints/phase2_v2.phase.pt` ✓ (ResonanceField + WaveToField + FieldToWave)

## What Changed vs Legacy Phase 9.5

| Aspect | Legacy Phase 9.5 | v2 Phase 3 |
|--------|-----------------|------------|
| Decode loss | None at training time | From step 1 |
| Imports | `phases/phase1` ... `phases/phase9` | `v2/phase1`, `v2/phase2` only |
| Context source | Full FLUXModel (phases 1–8 frozen) | Phase 1 + 2 checkpoints only |
| Generation gate | Checked only at Phase 9 | Checked every 1,000 steps |
| Context collapse fix | LayerNorm + L2-norm (applied) | Same + contrastive loss |

## Architecture

```
Prompt text
    │
    ▼
ContinuousSemanticEncoder (frozen, Phase 1)
    │  [seq_len, 432] waves
    ▼
mean_pool → [432]
    │
    ▼
WaveToField (frozen, Phase 2)
    │  [512] field context
    ▼
WaveGenerator.generate()
    │
    │  context [512]
    │      → LayerNorm + projection (decorrelation)
    │      → context_to_hidden → GRU initial hidden state h₀
    │      → context_to_wave → context_wave [432]
    │
    │  GRU (hidden=512, batch_first=True)
    │  input: cat[prev_wave [432], context_wave [432]] = [864]
    │  output: [432] next wave
    │
    ▼  (repeat N times)
Generated waves [N, 432]
    │
    ▼
WaveToText.decode() per wave (frozen, Phase 1)
    │
    ▼
Output text bytes
```

## Training

### Frozen components
- CSE, WaveChunker, WaveToText — from `phase1_v2.phase.pt`
- WaveToField, FieldToWave — from `phase2_v2.phase.pt`

### Trainable
- WaveGenerator only (~1.5M parameters)

### Loss
```
total = wave_loss         (MSE predicted vs target wave)
      + cosine_loss       (1 - cosine_similarity)
      + contrastive_loss  (penalise identical outputs for different contexts)
      + decode_loss       (cross-entropy WTT(predicted_wave) vs gt_bytes)
                          ↑ THIS IS THE KEY FIX — decode loss from step 1
```

### Scheduled Sampling
- Steps 0 → end: ss_p ramps from 50% → 90%
- 50% = use own prediction as next input (forces realistic generation distribution)
- 90% = almost fully autoregressive by end of training

### Decode Gate (checked every 1,000 steps)
```
avg byte accuracy  > 90%  → training passes gate
min byte accuracy  > 70%  → no single text completely fails
```

## Acceptance Criteria

- [ ] WaveGenerator trains without NaN losses
- [ ] Decode gate at training: avg ≥ 90%, min ≥ 70%
- [ ] Test 1: Generated byte accuracy avg > 50%, min > 20%
- [ ] Test 2: Inter-prompt avg cosine < 0.85 (no context collapse)
- [ ] Test 3: Valid word rate > 50% across 20 prompts
- [ ] Demo 1 runs and prints decoded output for all 8 gate texts
- [ ] Demo 2 runs and saves demo_phase3_demo2.png
- [ ] Checkpoint saved to `checkpoints/phase3_v2.phase.pt`
- [ ] Checkpoint uploaded to HuggingFace Hub (`UnseenGAP/FLUX`)
- [ ] Log written to `logs/phase3.log`

## Why the Decode Loss Is the Only Real Change

The legacy WaveGeneratorV3 architecture is solid. The GRU with:
- context decorrelation (LayerNorm + L2 normalize)
- batch_first=True for efficiency
- scheduled sampling from 50% on day 1
- contrastive loss to prevent context collapse

...all worked correctly in Phase 9.5. The ONLY reason it failed at Phase 9
was that `wave_to_field` and `field_to_wave` were random initialization —
they mapped waves to garbage and garbage back to waves. The generator had
no way to produce decodable output because the path from generator → WTT
was broken at the projection layer.

Phase 2 v2 fixed the projections. Phase 3 v2 adds decode_loss to make
the generator aware of that path from step 1. Everything else is the same.

## Expected Training Behavior

| Step | Expected |
|------|---------|
| 500 | wave_loss ~1.0, decode_loss ~3.0, gate ~5% |
| 2,000 | wave_loss ~0.3, decode_loss ~1.5, gate ~30% |
| 5,000 | wave_loss ~0.15, decode_loss ~0.8, gate ~60% |
| 10,000 | wave_loss ~0.08, decode_loss ~0.4, gate ~85% |
| 15,000-20,000 | gate passes (avg ≥ 90%) → early stop |

Total steps: 15,000–20,000 on an NVIDIA L4 (same GPU as Phases 1 and 2).
Estimated wall time: ~45–90 minutes.
